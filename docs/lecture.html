Part 1: The Evolution

Classical vs. Intelligent Systems

Deterministic Predictable. Static.

Stochastic Adaptive. Learning.

Script: “Historically, SE was about building clockwork. If I turn gear A, gear B moves. Today, we build systems like the Mars Rover. It encounters unknown terrain. We can’t pre-program every reaction. It has to learn.”

Why We Need AI

  • Data Volume: Terabytes/hour.
  • Dimensionality: Systems with 1000+ variables.
  • Human Limits: We cannot manually optimize these anymore.

Part 2: The Engineer’s Definition

Disentangling the Buzzwords

Script: “Let’s be precise. AI is the machine acting smart. ML is the algorithm learning from data. Deep Learning is the neural network approach. Data Science is the analysis of the logs.”

The Engineering View

Classical

\[ y = f(x) \] We know the physics (F=ma). We write the equation.

Machine Learning

\[ y \approx \hat{f}(x) \] We have the data points. The computer finds the curve.

Script: “To an engineer, ML is just ‘Function Approximation.’ We have a bunch of dots (data), and we ask the computer to draw the best line through them. That line becomes our law.”

Part 3: The Lifecycle

The V-Model and AI

Script: “This is the Holy Grail of SE: The V-Model. Think of it as the ‘Map of Engineering.’

  • The Basic V: We go down into the details (Left) and then back up into the real world (Right).
  • The AI Connection (Dual-Track): AI is just a subsystem. The V-Model ensures the ‘Smart’ AI code actually integrates with the ‘Real’ hardware.
  • The Requirement Shift: Instead of writing rules (If-Then), we design the Data. We are engineering the environment the AI learns from.
  • The Verification Gap: We can’t test every line of code in a neural network. Verification shifts from ‘Does it match logic?’ to ‘Does it handle the statistical edge cases?’”

Operations: The Digital Twin

Physical Asset

Digital Replica

Script: “We build a Digital Twin. The physical engine sends data to the digital one. The digital one uses ML to predict failures before they happen.”

Astria: Digital Twin

  • The Physical Asset: Autonomous drone with a gripper.
  • The Digital Replica: High-fidelity physics-based simulation.
  • Systems Goal: Testing contact physics and control logic virtually before risking expensive hardware on power lines.

Real-world (Left) vs. Sim2Real Replica (Right)

Script: “Here is a concrete example from our research. We have the physical Astria drone, and we have its high-fidelity digital replica. Before we fly near high-voltage lines, we test every control algorithm in the digital twin to ensure the system behaves as expected.”

Rimless Wheel: Robustness

  • Systems Goal: Walking that doesn’t fall when the ground changes.
  • Non-examples:
    • Falling over (unstable).
    • Tripping on uneven ground (fragile).
  • Examples:
    • Steady rhythm on flat ground.
    • Robustness: Recovering from uneven terrain.

How AI Learns to Control

Reinforcement Learning

  1. Observe state.
  2. Take Action (Voltage).
  3. Get Reward (+1 if upright).

Script: “We use Reinforcement Learning. The AI tries random things. It fails. It learns. Eventually, it masters the physics.”

Sim2Real: The Reality Check

Simulation

Perfect Math

Reality

Noise & Friction

Script: “We train in simulation (perfect). We deploy to reality (messy). If the model overfits, the robot crashes. We need Robust Control.”

Learning to Walk: RL in Action

  • The Goal: Make the robot walk forward.
  • The Process:
    • Start with zero knowledge.
    • Try random actions.
    • Get rewarded for forward motion.
    • Result: Emergent walking behavior.

Part 5: The Future

The Black Box Problem

Classical Code

IF speed > 50 THEN brake (Traceable)

Neural Network

0.23 * x1 + 0.99 * x2 ... (Opaque)

Script: “Traditional code is readable. Neural networks are millions of opaque numbers. Traceability is a massive open problem in SE.”
